# Registros de execução dos requisitos

Este diretório registra o que foi efetivamente implementado em cada fase do
[plano de ação do MVP](../09-plano-de-acao.md). O formato segue o padrão do projeto
canarinho2.0: cada etapa informa estado, escopo entregue, evidências, validações e pendências, sem
confundir intenção do plano com funcionalidade comprovada.

## Significado dos estados

| Estado | Critério |
| --- | --- |
| Não iniciada | Não há entrega funcional da fase no código ou no banco. |
| Em andamento | Há implementação parcial, mas ainda faltam entregas centrais. |
| Em fechamento | As capacidades centrais existem, porém ainda há critérios obrigatórios pendentes. |
| Concluída | Entregas e critérios de aceite foram verificados com evidências reproduzíveis. |

Ao atualizar uma fase, registre a data, os comandos realmente executados e as limitações conhecidas.
Não marque uma etapa como concluída apenas porque arquivos ou scripts foram criados.
